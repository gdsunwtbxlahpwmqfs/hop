## 1. 概述

### 1.1 什么是 Hop Web

Qi Hop Web 是 Hop Gui 的浏览器版本，基于 **Eclipse RAP（Remote Application Platform）** 技术，将桌面 SWT 界面转换为 Web 应用。它在功能上与桌面客户端保持一致，用户通过浏览器即可完成数据管道（Pipeline）和工作流（Workflow）的编排与执行。

### 1.2 运行架构

```
┌─────────────────────────────────────────────────────┐
│                   浏览器客户端                        │
│              http://<server-ip>:8080/ui              │
└──────────────────────┬──────────────────────────────┘
                       │ HTTP / WebSocket
┌──────────────────────▼──────────────────────────────┐
│                  Tomcat 10 容器                       │
│  ┌─────────────────────────────────────────────────┐│
│  │          webapps/ROOT  (Hop Web 应用)            ││
│  │  ┌───────────┐ ┌──────────┐ ┌────────────────┐  ││
│  │  │ RWT Servlet│ │HopServer │ │ Upload/Download│  ││
│  │  │  (/ui)    │ │(/hop/*)  │ │  / WebSocket    │  ││
│  │  └───────────┘ └──────────┘ └────────────────┘  ││
│  │            WEB-INF/lib (核心库)                   ││
│  └─────────────────────────────────────────────────┘│
│  ┌──────────┐  ┌──────────────┐  ┌───────────────┐  │
│  │ plugins/ │  │ jdbc-drivers/│  │  config/      │  │
│  │ (插件目录)│  │ (数据库驱动) │  │ (配置/项目)    │  │
│  └──────────┘  └──────────────┘  └───────────────┘  │
└─────────────────────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│                   JDK 21 运行时                       │
└─────────────────────────────────────────────────────┘
```

### 1.3 关键技术版本

| 组件     | 版本要求           | 说明                                 |
| ------ | -------------- | ---------------------------------- |
| JDK    | **21**（必须）     | Hop 基于 Java 21 编译，低版本无法运行          |
| Tomcat | **10.x**（必须）   | 使用 Jakarta EE 命名空间，Tomcat 9 及以下不兼容 |
| 操作系统   | Linux x86\_64  | 推荐 CentOS 7+/Ubuntu 18.04+/RHEL 8+ |
| 内存     | 最低 4GB，推荐 8GB+ | Hop 是内存密集型应用                       |
| 磁盘     | 最低 5GB 可用空间    | 包含日志、审计、项目文件                       |

## 2. 环境准备

### 2.1 系统要求检查

在目标服务器上执行以下检查：

```bash
# 检查操作系统
cat /etc/os-release

# 检查系统架构（必须为 x86_64）
uname -m

# 检查可用内存（至少 4GB）
free -h

# 检查可用磁盘空间（至少 5GB）
df -h /

# 检查端口 8080 是否被占用
ss -tlnp | grep 8080
```

### 2.2 创建系统用户

为安全起见，使用非 root 用户运行 Tomcat：

```bash
# 创建 hop 用户组及用户
sudo groupadd -r hop
sudo useradd -d /opt/hop -m -s /bin/bash -g hop hop

# 设置密码（可选）
sudo passwd hop
```

### 2.3 创建目录结构

```bash
# 基础安装目录
sudo mkdir -p /opt/hop
sudo chown -R hop:hop /opt/hop

# 切换到 hop 用户操作
sudo su - hop

# 创建子目录
mkdir -p /opt/hop/{tomcat,app,plugins,jdbc-drivers,config,audit,logs,project,data}
```

最终目录结构如下：

```
/opt/hop/
├── tomcat/          # Tomcat 安装目录
├── app/             # Hop Web 应用（ROOT）部署目录
├── plugins/         # Hop 插件目录
├── jdbc-drivers/    # JDBC 驱动目录
├── config/          # Hop 配置文件目录
├── audit/           # 审计日志目录
├── logs/            # 应用日志目录
├── project/         # 项目文件目录
└── data/            # 数据/元数据存储目录
```

## 3. 离线安装包清单与制作

> **核心原则**：在有网络的机器上准备好所有依赖，打包传输到目标服务器。

### 3.1 安装包清单

| 序号 | 组件          | 文件名                              | 获取方式                                     |
| -- | ----------- | -------------------------------- | ---------------------------------------- |
| 1  | JDK 21      | `jdk-21_linux-x64_bin.tar.gz`    | Oracle 官网 或 OpenJDK（推荐 Adoptium/Temurin） |
| 2  | Tomcat 10   | `apache-tomcat-10.1.x.tar.gz`    | Tomcat 官网 <https://tomcat.apache.org>    |
| 3  | Hop Client  | `hop-client-xxx.zip`             | Hop 构建：`assemblies/client/target/`       |
| 4  | Hop Web WAR | `hop.war`                        | Hop 构建：`assemblies/web/target/`          |
| 5  | Hop Plugins | `hop-assemblies-plugins-xxx.zip` | Hop 构建：`assemblies/plugins/target/`      |
| 6  | JDBC 驱动     | 各数据库 `jdbc-drivers/*.jar`                | 各数据库厂商官网                                 |
| 7  | 安装脚本        | `install-hop-web.sh`             | 见本文第 9 章                                 |

### 3.2 在有网络环境中构建 Hop 产物

如果从源码构建 Qi Hop：

```bash
# 克隆代码
git clone https://github.com/apache/hop.git
cd hop

# 快速构建（跳过测试，生成所有发布包）
./mvnw clean install -Pfast-build -DskipTests -Dassemblies=false

# 或者完整构建
./mvnw clean install -Pfast-build -DskipTests
```

构建产物位置：

```bash
# Hop Web WAR 包
assemblies/web/target/hop.war

# Hop Client 压缩包（包含核心库、插件）
assemblies/client/target/hop-client-*.zip

# Hop 插件包
assemblies/plugins/target/hop-assemblies-plugins-*.zip
```

### 3.3 制作离线安装包

在有网络的机器上，将所有文件组织到统一目录并打包：

```bash
mkdir -p ~/hop-offline-pkg/{jdk,tomcat,hop,scripts,jdbc-drivers}

# 1. 放入 JDK 21
cp jdk-21_linux-x64_bin.tar.gz ~/hop-offline-pkg/jdk/

# 2. 放入 Tomcat
cp apache-tomcat-10.1.x.tar.gz ~/hop-offline-pkg/tomcat/

# 3. 放入 Hop 产物
cp hop/assemblies/web/target/hop.war ~/hop-offline-pkg/hop/
cp hop/assemblies/client/target/hop-client-*.zip ~/hop-offline-pkg/hop/
cp hop/assemblies/plugins/target/hop-assemblies-plugins-*.zip ~/hop-offline-pkg/hop/

# 4. 放入 JDBC 驱动
cp mysql-connector-j-*.jar ~/hop-offline-pkg/jdbc-drivers/
cp ojdbc*.jar ~/hop-offline-pkg/jdbc-drivers/
cp postgresql-*.jar ~/hop-offline-pkg/jdbc-drivers/

# 5. 放入安装脚本（见第 9 章）
cp install-hop-web.sh ~/hop-offline-pkg/scripts/

# 6. 打包
cd ~
tar -czf hop-offline-pkg.tar.gz hop-offline-pkg/
```

### 3.4 传输到目标服务器

通过 U 盘、内网 SCP 或其他离线方式传输：

```bash
# 示例：通过 SCP 传输（如果内网可达）
scp hop-offline-pkg.tar.gz user@<目标服务器IP>:/tmp/

# 在目标服务器上解压
cd /tmp
tar -xzf hop-offline-pkg.tar.gz
```

## 4. 安装步骤

### 4.1 安装 JDK 21

```bash
# 解压 JDK
cd /opt/hop
tar -xzf /tmp/hop-offline-pkg/jdk/jdk-21_linux-x64_bin.tar.gz

# 重命名目录（简化路径）
mv jdk-21.* jdk21

# 验证安装
/opt/hop/jdk21/bin/java -version
```

预期输出：

```
openjdk version "21.x.x" 20xx-xx-xx
OpenJDK Runtime Environment ...
OpenJDK 64-Bit Server VM ...
```

### 4.2 安装 Tomcat 10

```bash
# 解压 Tomcat
cd /opt/hop
tar -xzf /tmp/hop-offline-pkg/tomcat/apache-tomcat-10.1.x.tar.gz

# 重命名目录
mv apache-tomcat-10.1.* tomcat

# 清除 Tomcat 默认 Web 应用（避免安全风险）
rm -rf /opt/hop/tomcat/webapps/*

# 创建 ROOT 目录（Hop Web 部署位置）
mkdir -p /opt/hop/tomcat/webapps/ROOT

# 设置权限
chown -R hop:hop /opt/hop/tomcat
```

### 4.3 部署 Hop Web 应用

> **重要提示**：`hop.war` 本身不包含 `hop-core`、`hop-engine` 等核心运行库。必须从 Client 包中提取核心库补充到 `WEB-INF/lib`，否则 Tomcat 启动会报 `NoClassDefFoundError`。

```bash
WEBAPP_DIR="/opt/hop/tomcat/webapps/ROOT"
PKG_DIR="/tmp/hop-offline-pkg/hop"

# 1. 解压 hop.war 到 ROOT 目录
cd "$WEBAPP_DIR"
unzip -q "$PKG_DIR/hop.war" -d "$WEBAPP_DIR"

# 2. 从 Client 包提取核心库（hop-core, hop-engine 等）
CLIENT_ZIP=$(ls "$PKG_DIR"/hop-client-*.zip | head -1)
WORK_DIR="/tmp/hop-client-extract"
rm -rf "$WORK_DIR"
mkdir -p "$WORK_DIR"
unzip -q "$CLIENT_ZIP" -d "$WORK_DIR"

# 3. 复制核心库到 WEB-INF/lib
cp "$WORK_DIR"/hop/lib/core/*.jar "$WEBAPP_DIR/WEB-INF/lib/"
cp "$WORK_DIR"/hop/lib/beam/*.jar "$WEBAPP_DIR/WEB-INF/lib/"

# 4. 删除冲突的 RCP fragment（hop-ui-rcp 与 hop-ui-rap 冲突）
rm -f "$WEBAPP_DIR/WEB-INF/lib"/hop-ui-rcp*

# 5. 确保 SWT 库与目标操作系统匹配
#    删除可能存在的 macOS cocoa 版本，保留 Linux GTK 版本
rm -f "$WEBAPP_DIR/WEB-INF/lib"/org.eclipse.swt.cocoa.*
# 如果 GTK 版本缺失，从 Client 包复制
if ! ls "$WEBAPP_DIR/WEB-INF/lib"/org.eclipse.swt.gtk.linux.* >/dev/null 2>&1; then
    cp "$WORK_DIR"/hop/lib/core/org.eclipse.swt.gtk.linux.*.jar "$WEBAPP_DIR/WEB-INF/lib/" 2>/dev/null || true
fi

# 6. 复制 Hop 配置文件到 config 目录
cp -r "$WORK_DIR"/hop/config/* /opt/hop/config/

# 7. 部署插件
PLUGINS_ZIP=$(ls "$PKG_DIR"/hop-assemblies-plugins-*.zip | head -1)
rm -rf /tmp/hop-plugins-extract
mkdir -p /tmp/hop-plugins-extract/tmp
unzip -q "$PLUGINS_ZIP" -d /tmp/hop-plugins-extract/tmp
mv /tmp/hop-plugins-extract/tmp/plugins/* /opt/hop/plugins/ 2>/dev/null || true
rm -rf /tmp/hop-plugins-extract

# 8. 部署 JDBC 驱动
cp /tmp/hop-offline-pkg/jdbc-drivers/*.jar /opt/hop/jdbc-drivers/ 2>/dev/null || true

# 9. 复制 Hop CLI 脚本（hop-run.sh, hop-conf.sh 等）到应用目录
cp "$WORK_DIR"/hop/*.sh "$WEBAPP_DIR/" 2>/dev/null || true
chmod +x "$WEBAPP_DIR"/*.sh 2>/dev/null || true

# 10. 清理临时文件
rm -rf "$WORK_DIR"

# 11. 设置权限
chown -R hop:hop /opt/hop
```

### 4.4 修正 hop-config.json 路径

```bash
# 将配置中的相对路径替换为绝对路径
sed -i 's|config/projects|/opt/hop/config/projects|g' \
    /opt/hop/config/hop-config.json
```

### 4.5 修正 CLI 脚本 Classpath

Hop 的 CLI 脚本默认假设从 Client 目录运行，部署到 Tomcat 后需要修正 classpath：

```bash
WEBAPP_DIR="/opt/hop/tomcat/webapps/ROOT"
for script in hop-run.sh hop-conf.sh hop-search.sh hop-encrypt.sh hop-import.sh; do
    if [ -f "$WEBAPP_DIR/$script" ]; then
        sed -i 's&lib/core/\*&../../lib/*:WEB-INF/lib/*:lib/core/*&g' "$WEBAPP_DIR/$script"
    fi
done
```

## 5. 配置说明

### 5.1 Tomcat 环境变量配置

创建 `/opt/hop/tomcat/bin/setenv.sh`（Tomcat 启动时自动加载）：

```bash
cat > /opt/hop/tomcat/bin/setenv.sh << 'ENVEOF'
#!/bin/bash
# =============================================================
# Hop Web Tomcat 环境变量配置
# =============================================================

# Java 虚拟机路径
export JRE_HOME="/opt/hop/jdk21"
export JAVA_HOME="/opt/hop/jdk21"

# Hop 核心配置
export HOP_CONFIG_FOLDER="/opt/hop/config"
export HOP_AUDIT_FOLDER="/opt/hop/audit"
export HOP_LOG_LEVEL="Basic"
export HOP_PASSWORD_ENCODER_PLUGIN="Hop"
export HOP_PLUGIN_BASE_FOLDERS="/opt/hop/plugins"
export HOP_SHARED_JDBC_FOLDERS="/opt/hop/jdbc-drivers"
export HOP_GUI_ZOOM_FACTOR="1.0"
export HOP_AES_ENCODER_KEY=""

# 项目配置（可选，按需修改）
export HOP_PROJECT_FOLDER=""
export HOP_PROJECT_NAME=""
export HOP_PROJECT_CONFIG_FILE_NAME="project-config.json"
export HOP_ENVIRONMENT_NAME="environment1"
export HOP_ENVIRONMENT_CONFIG_FILE_NAME_PATHS=""

# CATALINA_OPTS：将 Hop 变量传递为 JVM 系统属性
CATALINA_OPTS="${HOP_OPTIONS}"
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_CONFIG_FOLDER=\"${HOP_CONFIG_FOLDER}\""
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_AUDIT_FOLDER=\"${HOP_AUDIT_FOLDER}\""
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_LOG_LEVEL=\"${HOP_LOG_LEVEL}\""
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_PASSWORD_ENCODER_PLUGIN=\"${HOP_PASSWORD_ENCODER_PLUGIN}\""
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_PLUGIN_BASE_FOLDERS=\"${HOP_PLUGIN_BASE_FOLDERS}\""
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_SHARED_JDBC_FOLDERS=\"${HOP_SHARED_JDBC_FOLDERS}\""
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_GUI_ZOOM_FACTOR=\"${HOP_GUI_ZOOM_FACTOR}\""
CATALINA_OPTS="${CATALINA_OPTS} -DHOP_AES_ENCODER_KEY=\"${HOP_AES_ENCODER_KEY}\""

# JVM 性能参数
# AggressiveHeap 让 JVM 自动使用所有可用内存
CATALINA_OPTS="${CATALINA_OPTS} -XX:+AggressiveHeap"
# RAP 资源缓存目录（避免每次请求重新生成）
CATALINA_OPTS="${CATALINA_OPTS} -Dorg.eclipse.rap.rwt.resourceLocation=/tmp/rwt-resources"
# 时区与编码
CATALINA_OPTS="${CATALINA_OPTS} -Duser.timezone=UTC -Dfile.encoding=UTF-8"

# Java 模块开放（Java 9+ 模块系统兼容性）
CATALINA_OPTS="${CATALINA_OPTS} --add-opens=java.base/java.lang=ALL-UNNAMED"
CATALINA_OPTS="${CATALINA_OPTS} --add-opens=java.base/java.net=ALL-UNNAMED"
CATALINA_OPTS="${CATALINA_OPTS} --add-opens=java.base/java.io=ALL-UNNAMED"
CATALINA_OPTS="${CATALINA_OPTS} --add-opens=java.base/java.util=ALL-UNNAMED"
CATALINA_OPTS="${CATALINA_OPTS} --add-opens=java.base/java.nio=ALL-UNNAMED"
CATALINA_OPTS="${CATALINA_OPTS} --add-opens=java.base/sun.nio.ch=ALL-UNNAMED"
CATALINA_OPTS="${CATALINA_OPTS} --add-opens=java.xml/com.sun.org.apache.xerces.internal.parsers=ALL-UNNAMED"

export CATALINA_OPTS

# 日志输出目录
export CATALINA_OUT="/opt/hop/logs/catalina.out"

ENVEOF

chmod +x /opt/hop/tomcat/bin/setenv.sh
```

### 5.2 环境变量详解

| 变量名                                      | 说明                                                           | 默认值                     |
| ---------------------------------------- | ------------------------------------------------------------ | ----------------------- |
| `HOP_CONFIG_FOLDER`                      | Hop 全局配置文件目录（hop-config.json）                                | `/opt/hop/config`       |
| `HOP_AUDIT_FOLDER`                       | 审计日志存储目录                                                     | `/opt/hop/audit`        |
| `HOP_LOG_LEVEL`                          | 日志级别：`Error`/`Minimal`/`Basic`/`Detailed`/`Debug`/`Rowlevel` | `Basic`                 |
| `HOP_PLUGIN_BASE_FOLDERS`                | 插件扫描目录（逗号分隔）                                                 | `/opt/hop/plugins`      |
| `HOP_SHARED_JDBC_FOLDERS`                | JDBC 驱动目录                                                    | `/opt/hop/jdbc-drivers` |
| `HOP_PASSWORD_ENCODER_PLUGIN`            | 密码编码器插件                                                      | `Hop`                   |
| `HOP_GUI_ZOOM_FACTOR`                    | 界面缩放系数                                                       | `1.0`                   |
| `HOP_PROJECT_NAME`                       | 启动时自动创建的项目名                                                  | 空（不创建）                  |
| `HOP_PROJECT_FOLDER`                     | 项目主目录路径                                                      | 空                       |
| `HOP_ENVIRONMENT_NAME`                   | 环境名称                                                         | `environment1`          |
| `HOP_ENVIRONMENT_CONFIG_FILE_NAME_PATHS` | 环境配置文件路径（逗号分隔）                                               | 空                       |

### 5.3 Tomcat 服务器配置（可选）

#### 5.3.1 修改端口（如需修改默认 8080）

编辑 `/opt/hop/tomcat/conf/server.xml`：

```xml
<!-- 将 8080 改为其他端口，如 9090 -->
<Connector port="9090" protocol="HTTP/1.1"
           connectionTimeout="20000"
           redirectPort="8443"
           maxParameterCount="1000"
           URIEncoding="UTF-8" />
```

#### 5.3.2 配置 Basic 认证（推荐生产环境启用）

**步骤一**：创建 `/opt/hop/tomcat/conf/tomcat-users.xml`

```xml
<?xml version='1.0' encoding='utf-8'?>
<tomcat-users>
  <role rolename="apachehop"/>
  <user username="admin" password="YourStrongPassword" roles="apachehop" />
</tomcat-users>
```

**步骤二**：修改 `/opt/hop/tomcat/webapps/ROOT/WEB-INF/web.xml`，在末尾 `</web-app>` 之前添加安全约束：

```xml
    <security-constraint>
      <web-resource-collection>
        <web-resource-name>HopWeb Protected</web-resource-name>
        <url-pattern>/*</url-pattern>
        <http-method>GET</http-method>
        <http-method>POST</http-method>
      </web-resource-collection>
      <auth-constraint>
        <role-name>apachehop</role-name>
      </auth-constraint>
      <user-data-constraint>
        <transport-guarantee>NONE</transport-guarantee>
      </user-data-constraint>
    </security-constraint>

    <login-config>
      <auth-method>BASIC</auth-method>
    </login-config>
```

### 5.4 JVM 内存调优建议

根据服务器内存大小调整 `CATALINA_OPTS` 中的堆参数：

| 服务器内存 | 推荐配置                          |
| ----- | ----------------------------- |
| 4GB   | `-Xms2g -Xmx2g`               |
| 8GB   | `-Xms4g -Xmx4g`               |
| 16GB  | `-XX:+AggressiveHeap`（自动分配）   |
| 32GB+ | `-Xms8g -Xmx12g -XX:+UseG1GC` |

> 如果使用 `-XX:+AggressiveHeap`，JVM 会自动使用容器/服务器分配的全部内存。在物理机部署中，建议明确指定 `-Xmx` 以避免占满系统内存。

## 6. 服务启动与验证

```bash

sudo su - hop

/opt/hop/tomcat/bin/catalina.sh run
```

```
INFO [main] org.apache.catalina.startup.Catalina.start Server startup in [xxxx] milliseconds
```

```bash
# 检查 Tomcat 进程
```

### 6.4 浏览器访问验证

打开浏览器访问以下地址：

| 功能                      | URL                               |
| ----------------------- | --------------------------------- |

## 8. 常见问题解决

### 8.1 启动报 `NoClassDefFoundError: HopException`

**原因**：`hop.war` 不包含核心库（hop-core/hop-engine），`WEB-INF/lib` 缺少核心 JAR。

**解决**：从 Client 包提取核心库并复制：

```bash
# 确认核心库是否已复制
ls /opt/hop/tomcat/webapps/ROOT/WEB-INF/lib/hop-core*.jar
ls /opt/hop/tomcat/webapps/ROOT/WEB-INF/lib/hop-engine*.jar

# 如果缺失，从 Client 包重新提取
CLIENT_ZIP="/tmp/hop-offline-pkg/hop/hop-client-*.zip"
unzip -q "$CLIENT_ZIP" "hop/lib/core/*" -d /tmp/hop-fix
cp /tmp/hop-fix/hop/lib/core/*.jar \
   /opt/hop/tomcat/webapps/ROOT/WEB-INF/lib/
chown -R hop:hop /opt/hop/tomcat/webapps/ROOT/WEB-INF/lib/
```

### 8.2 启动报 `ClassNotFoundException: ... SWT ...`

**原因**：SWT 库与操作系统不匹配（如包含 macOS cocoa 版本而非 Linux GTK 版本）。

**解决**：

```bash
WEBAPP_LIB="/opt/hop/tomcat/webapps/ROOT/WEB-INF/lib"

# 删除 macOS 版本
rm -f "$WEBAPP_LIB"/org.eclipse.swt.cocoa.*

# 确认 Linux GTK 版本存在
ls "$WEBAPP_LIB"/org.eclipse.swt.gtk.linux.*

# 如果缺失，从 Client 包复制
CLIENT_ZIP="/tmp/hop-offline-pkg/hop/hop-client-*.zip"
unzip -q "$CLIENT_ZIP" "hop/lib/core/org.eclipse.swt.gtk*" -d /tmp/hop-swt
cp /tmp/hop-swt/hop/lib/core/org.eclipse.swt.gtk.linux.*.jar "$WEBAPP_LIB/"
chown hop:hop "$WEBAPP_LIB"/org.eclipse.swt.gtk.*
```

### 8.3 启动报 `TextSizeUtilFacadeImpl` 冲突

**原因**：`hop-ui-rcp` fragment 与 `hop-ui-rap` 冲突（两者提供相同的 facade 实现）。

**解决**：

```bash
rm -f /opt/hop/tomcat/webapps/ROOT/WEB-INF/lib/hop-ui-rcp*
```

### 8.4 页面白屏 / RWT 资源加载失败

**原因**：RAP 资源缓存目录权限不足或不存在。

**解决**：

```bash
# 创建并设置 RWT 资源目录
sudo mkdir -p /tmp/rwt-resources
sudo chown -R hop:hop /tmp/rwt-resources
sudo chmod 755 /tmp/rwt-resources

# 确认 setenv.sh 中有此配置
grep "resourceLocation" /opt/hop/tomcat/bin/setenv.sh
```

### 8.5 端口 8080 被占用

**解决**：

```bash
# 查看占用进程
ss -tlnp | grep 8080
# 或
lsof -i:8080

# 方案一：停止占用进程
sudo kill -9 <PID>

# 方案二：修改 Hop Web 端口（编辑 server.xml）
# 将 Connector port="8080" 改为其他端口
```

### 8.6 插件未加载 / 数据库连接不可用

**原因**：`HOP_PLUGIN_BASE_FOLDERS` 或 `HOP_SHARED_JDBC_FOLDERS` 路径配置错误。

**排查步骤**：

```bash
# 检查环境变量是否正确传递
# 在 setenv.sh 中添加临时调试输出
echo "HOP_PLUGIN_BASE_FOLDERS=$HOP_PLUGIN_BASE_FOLDERS" >> /opt/hop/logs/debug.log
echo "HOP_SHARED_JDBC_FOLDERS=$HOP_SHARED_JDBC_FOLDERS" >> /opt/hop/logs/debug.log

# 确认目录中有内容
ls /opt/hop/plugins/ | head -20
ls /opt/hop/jdbc-drivers/

# 确认 catalina.out 中是否有插件加载日志
grep -i "plugin" /opt/hop/logs/catalina.out | head -20
```

### 8.7 内存溢出（OutOfMemoryError）

**原因**：JVM 堆内存不足。

**解决**：编辑 `setenv.sh`，增大堆内存：

```bash
# 编辑 setenv.sh，替换内存配置
# 移除 AggressiveHeap，改为明确指定
CATALINA_OPTS="${CATALINA_OPTS} -Xms4g -Xmx4g -XX:+UseG1GC"
CATALINA_OPTS="${CATALINA_OPTS} -XX:MaxGCPauseMillis=200"
CATALINA_OPTS="${CATALINA_OPTS} -XX:+HeapDumpOnOutOfMemoryError"
CATALINA_OPTS="${CATALINA_OPTS} -XX:HeapDumpPath=/opt/hop/logs/"
```

### 8.8 防火墙阻止访问

**CentOS/RHEL（firewalld）**：

```bash
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload
```

**Ubuntu（ufw）**：

```bash
sudo ufw allow 8080/tcp
sudo ufw reload
```

### 8.9 Tomcat 启动很慢（超过 2 分钟）

**原因**：Tomcat 的 `SessionIdGenerator` 使用 `/dev/random`，在熵不足时阻塞。

**解决**：编辑 `setenv.sh`，添加：

```bash
CATALINA_OPTS="${CATALINA_OPTS} -Djava.security.egd=file:/dev/./urandom"
```

## 9. 自动化安装脚本

安装脚本已提取为独立文件，详见 [install-hop-web.sh](install-hop-web.sh)。

该脚本一键完成以下操作：

| 步骤 | 说明 |
|------|------|
| 前置检查 | 校验 root 权限、离线包完整性、端口占用 |
| 用户与目录 | 创建 `hop` 用户及 `/opt/hop` 目录结构 |
| 安装 JDK 21 | 解压并重命名为 `${INSTALL_BASE}/jdk21` |
| 安装 Tomcat 10 | 解压并重命名为 `${INSTALL_BASE}/tomcat`，清理默认 webapp |
| 部署 Hop Web | 解压 `hop.war`、补充核心库、修正 SWT/CLI 脚本、部署插件与 JDBC 驱动 |
| 配置 setenv.sh | 生成 JVM 参数、Hop 环境变量、`--add-opens` 模块开放 |
| 配置 systemd | 生成 `hop-web.service`，设置开机自启 |
| 启动验证 | 启动服务并轮询 `/hop/status/` 探活 |

> 部署前请将脚本放入离线安装包的 `scripts/` 目录。

### 9.1 使用方法

```bash
# 1. 将离线安装包传输到目标服务器
scp hop-offline-pkg.tar.gz user@<服务器IP>:/tmp/

# 2. 在目标服务器解压
cd /tmp && tar -xzf hop-offline-pkg.tar.gz

# 3. 执行安装脚本
sudo bash /tmp/hop-offline-pkg/scripts/install-hop-web.sh

# 4. 自定义端口安装
sudo TOMCAT_PORT=9090 bash /tmp/hop-offline-pkg/scripts/install-hop-web.sh
```
