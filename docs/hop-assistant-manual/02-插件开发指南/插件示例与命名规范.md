# 插件示例与命名规范

## 功能概述


本指南介绍 Qi 数据治理平台插件示例的组织方式和命名规范，帮助开发者创建易于理解和维护的插件示例项目。

## 示例项目

Qi 数据治理平台发行版中的 `samples` 项目位于 `config/projects/samples` 文件夹中，其父项目是 `default` 项目。这意味着不需要向 `samples` 项目添加已存在于 `default` 项目中的元数据对象（如运行配置）。

## 添加插件示例

如果插件在 `src/main/` 文件夹下放置 `samples/` 文件夹，其内容将被拾取并复制到 `samples` 项目文件夹中，内容将以完全相同的文件夹结构存放。

### 示例路径映射

| 源路径 | 目标路径 |
|--------|----------|
| `plugins/transforms/addsequence/src/main/samples/transforms/add-sequence-add-a-unique-id.hpl` | `config/projects/samples/transforms/add-sequence-add-a-unique-id.hpl` |
| `plugins/engines/beam/src/main/samples/metadata/pipeline-run-configuration/Direct.json` | `config/projects/samples/metadata/pipeline-run-configuration/Direct.json` |

### 添加额外元数据

只需将额外元数据放入插件 `samples/` 文件夹下的 `metadata/` 文件夹即可轻松添加。测试数据 `datasets` 等同理。

## 标准项目内容

如果你有元数据对象、示例输入文件、数据集或希望对所有插件可用的文件，请将它们添加到源代码中的以下位置：

```
assemblies/static/src/main/resources/config/projects/samples
```

## 命名规范

命名示例和文件夹时请遵循以下基本规则：

| 规则 | 说明 |
|------|------|
| 文件命名 | 文件名以转换、动作或插件的名称开头 |
| 大小写 | 全部小写 |
| 分隔符 | 文件名中不要使用空格或下划线：使用连字符（-） |
| 文件夹组织 | 将插件放入正确的文件夹：转换示例放入 `transforms/`，动作示例放入 `actions/` |
| 输入文件 | 如果需要为示例添加输入文件，请将其放入 `files/` 子文件夹（如 `transforms/files`） |
| 最佳实践 | 请遵循最佳实践指南创建示例 |
| 注释说明 | 在管道和工作流中创建注释说明发生了什么，注释字体足够大（11 或更高） |

## XML配置示例

示例管道文件的结构示例（`.hpl` 文件）：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<pipeline>
  <info>
    <name>add-sequence-add-a-unique-id</name>
    <description>示例：使用 Add Sequence 添加唯一ID</description>
    <parameters>
    </parameters>
  </info>
  <order>
    <hop>
      <from>Generate Rows</from>
      <to>Add Sequence</to>
      <enabled>Y</enabled>
    </hop>
    <hop>
      <from>Add Sequence</from>
      <to>Write to Log</to>
      <enabled>Y</enabled>
    </hop>
  </order>
  <transform>
    <name>Generate Rows</name>
    <type>GenerateRows</type>
    <copies>1</copies>
    <partitioning>
      <method>none</method>
    </partitioning>
    <rows>5</rows>
  </transform>
  <transform>
    <name>Add Sequence</name>
    <type>AddSequence</type>
    <copies>1</copies>
    <partitioning>
      <method>none</method>
    </partitioning>
    <name_field>sequence_id</name_field>
    <start_at>1</start_at>
    <increment_by>1</increment_by>
  </transform>
  <transform>
    <name>Write to Log</name>
    <type>WriteToLog</type>
    <copies>1</copies>
    <partitioning>
      <method>none</method>
    </partitioning>
    <log_level>INFO</log_level>
    <subject>Row data</subject>
  </transform>
</pipeline>
```

## 注意事项

- 示例文件名应清晰描述其功能
- 每个示例应包含足够的注释说明
- 示例管道应可独立运行，不需要外部依赖
- 示例应遵循 Qi 数据治理平台的编码规范
