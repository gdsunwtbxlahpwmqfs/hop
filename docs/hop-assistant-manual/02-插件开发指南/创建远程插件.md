# 创建远程插件

## 功能概述


本指南帮助你设置并开始开发一个不纳入 Qi 数据治理平台仓库的独立插件。
Qi 数据治理平台对源代码许可证（包括用于提供特定功能的所有依赖项）有严格的政策。如果你的插件由 A 类（Category A）代码和依赖项组成，则可以安全地添加到 Qi 数据治理平台仓库中，社区可以帮助维护代码。如果使用了 B 类或 X 类，则不能再添加到 Qi 数据治理平台的默认发行版中。

## 入门步骤

本指南假设你使用 IntelliJ 进行开发，某些步骤在 Eclipse 中可能有所不同。

| 步骤 | 说明 |
|------|------|
| 1 | 克隆骨架仓库（hop-plugin-sample） |
| 2 | 在 IntelliJ 中导入项目 |
| 3 | 开始编码 |

## 项目结构

示例项目由 3 个主模块和 3 个子模块组成：

```
hop-action-sample/      — 动作示例
hop-transform-sample/   — 转换示例
assemblies/
  assemblies-action-sample/
  assemblies-transform-sample/
  debug/                 — 调试模块
```

在示例仓库中提供了一个动作和一个转换，它们都能正常工作，但不操作数据或执行任何实际动作。assemblies 模块用于创建发行版或在开发期间测试你的动作或转换。

## 调试配置

要调试项目，请创建一个新的 `application` 配置。重要的是工作目录——为了让 Qi 数据治理平台（包括项目提供的所有插件）正常工作，需要特定的文件夹结构。

### 核心依赖配置

```xml
<dependency>
    <groupId>org.apache.hop</groupId>
    <artifactId>hop-core</artifactId>
    <version>${hop.version}</version>
    <scope>compile</scope>
</dependency>
<dependency>
    <groupId>org.apache.hop</groupId>
    <artifactId>hop-engine</artifactId>
    <version>${hop.version}</version>
    <scope>compile</scope>
</dependency>
<dependency>
    <groupId>org.apache.hop</groupId>
    <artifactId>hop-ui-swt</artifactId>
    <version>${hop.version}</version>
    <scope>compile</scope>
</dependency>
<dependency>
    <groupId>org.apache.hop</groupId>
    <artifactId>hop-assemblies-plugins-dist</artifactId>
    <version>${hop.version}</version>
    <type>zip</type>
    <exclusions>
        <exclusion>
            <groupId>*</groupId>
            <artifactId>*</artifactId>
        </exclusion>
    </exclusions>
    <scope>compile</scope>
</dependency>
```

## XML配置示例

远程插件的 `pom.xml` 配置示例：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    
    <groupId>com.example</groupId>
    <artifactId>hop-transform-sample</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <packaging>jar</packaging>
    
    <parent>
        <groupId>org.apache.hop</groupId>
        <artifactId>hop-plugins-transforms</artifactId>
        <version>${hop.version}</version>
        <relativePath>../pom.xml</relativePath>
    </parent>
    
    <properties>
        <hop.version>1.0.0-SNAPSHOT</hop.version>
    </properties>
    
    <dependencies>
        <dependency>
            <groupId>org.apache.hop</groupId>
            <artifactId>hop-core</artifactId>
        </dependency>
        <dependency>
            <groupId>org.apache.hop</groupId>
            <artifactId>hop-engine</artifactId>
        </dependency>
    </dependencies>
</project>
```

## 注意事项

- 编译时调试位置会添加一个 plugins 目录，包含 Qi 数据治理平台团队发行的所有插件
- 示例转换和动作的组装包遵循相同的逻辑，确保它们也被复制到正确的位置
- 插件加载器能够找到并加载 GUI 所需的所有插件
