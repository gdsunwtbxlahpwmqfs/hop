# 元数据插件开发

在 Hop 中编写元数据插件非常简单，任何普通 Java 对象（POJO）都可作为起点。

## @HopMetadata 注解

此注解向元数据插件类型发出信号，表示有值得关注的内容。带有此注解的类将包含元数据。

`@HopMetadata` 注解的属性：

- **key**：唯一标识插件，也是序列化为 JSON 时元数据所在文件夹的名称
- **name**：人类可读的名称
- **description**：扩展描述
- **image**：帮助在 Hop GUI 中标识元数据的图像路径

带有此注解的类将因为位于 Hop 的 `plugins/` 文件夹中而被发现，或者如果它是内部类，则在 `engine/src/main/resources/hop-metadata-plugins.xml` 文件中描述。

## 元数据属性

所有希望成为共享 Hop 元数据一部分的属性都应添加 `@HopMetadataProperty` 注解。所有带有 `@HopMetadata` 的顶层类都应有一个 `String` 类型的 `name` 属性。

`@HopMetadataProperty` 属性：

- **key**：如果希望与字段名不同的可选键
- **password**：如果希望 String 字段使用 `IHopMetadataProvider` 接口的 `TwoWayPasswordEncoder` 进行编码，则设为 true
- **storeWithName**：如果希望存储对另一个共享元数据对象的引用，则设为 true，否则将存储该对象的所有属性

支持的数据类型：

- **enum**：任何枚举都使用其名称序列化
- **String**：另见上文的 password 属性
- **Integer / int**
- **Long / long**
- **Boolean / boolean**
- **java.util.Date**
- **java.util.Map<String,String>**
- **java.util.List<T>**：T 为此处列出的任何数据类型
- **POJO**：任何包含更多 `@HopMetadataProperty` 注解的类

## 编辑器类

你还需要一种在 GUI 中编辑元数据的方式，这可以通过继承 `MetadataEditor<IHopMetadata>` 类来实现。编辑器类的路径会通过查看元数据插件类的名称并简单追加 `Editor` 来自动发现。如果你更喜欢将元数据和 GUI 代码分开，Hop GUI 也会在 `org.apache.hop.ui` 包中查找（而不是 `org.apache.hop`）。

工作示例：
- `org.apache.hop.path.to.MyMetadata` → `org.apache.hop.path.to.MyMetadataEditor`
- `org.apache.hop.partition.PartitionSchema` → `org.apache.hop.ui.partition.PartitionSchemaEditor`

主要方法说明：
- `createControl()`：在给定的父 Composite 上创建各种控件
- `setWidgetsContent()`：使用元数据对象在已创建的控件上设置内容
- `getWidgetsContent()`：获取各控件的值并修改元数据
- `save()`：验证设置、获取元数据并调用 `super.save()`
- `setFocus()`：在元数据对话框或选项卡上设置焦点
- `createButtonsForButtonBar()`：在底部添加按钮以执行测试、查看等操作

## 元数据序列化

`@HopMetadata` 插件的 key 或 ID 用作存储对象的顶层文件夹。序列化为 JSON 时支持大多数简单数据类型，但建议遵循 KISS（保持简单）原则。如果要序列化接口（例如 DatabaseMeta 使用的 IDatabase），可能需要用 `@HopMetadataObject` 注解标记接口。此注解允许你为这些类指定对象工厂。这样的对象工厂实现 `IHopMetadataObjectFactory` 接口，包含以下两个方法：

- `createObject(String id, Object parentObject)`：使用 ID 创建对象。父对象通常是另一个元数据对象。
- `getObjectId(Object object)`：从给定对象检索对象 ID。
