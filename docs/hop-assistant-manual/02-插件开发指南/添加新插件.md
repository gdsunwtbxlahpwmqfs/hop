# 添加新插件

## 功能概述


本指南介绍如何在 Qi 数据治理平台中创建新插件，包括创建骨架、添加到组装包以及模块结构说明。

## 创建骨架

确定插件类别后，将插件创建为一个新的 Maven 子模块：在插件 pom 文件中仔细声明 parent，并在父项目的 pom 中添加新 module。这确保将新插件纳入 Maven 构建周期。

可以复制参考插件作为起点。复制时请务必修改文档并删除无关内容，以获得干净的骨架。

## 添加到组装包（Assemblies）

要将插件添加到最终构件中，需将其作为 Maven 子模块添加到 `hop/assemblies/plugins` 目录下对应的类别中：在插件 pom 文件中仔细声明 parent，并在父项目中添加新 module。

定义 `src/assembly/assembly.xml`，包含插件运行所需的必要依赖 jar。请审查最终进入插件 zip 包的 jar 文件。如果某个 jar 已存在于 hop/lib 目录中，则应将其排除。这样做是为了使目标 hop 构件尽可能小。

## 模块结构

```
core/            — 核心库（hop-core）
engine/          — 管道/工作流执行引擎
engine-beam/     — Apache Beam 引擎集成
lib/ lib-jdbc/ lib-p2/ — 共享库
plugins/
  transforms/    — 约 150 个数据转换插件
  actions/       — 约 55 个工作流动作插件
  databases/     — 数据库连接插件
  engines/       — 引擎插件（如 Beam）
  misc/ tech/ resolvers/ valuetypes/ — 其他插件类别
ui/              — 基于 SWT 的共享 UI 代码
rcp/             — 桌面应用程序
rap/             — Web 应用程序
rest/            — REST API 服务器
assemblies/      — 发行版打包
integration-tests/ — 集成测试套件
```

插件构件命名规范：`hop-<type>-<name>`（例如 `hop-transform-json`、`hop-action-ftp`）。

## 添加新插件时的注意事项

| 步骤 | 说明 |
|------|------|
| 1 | 在对应父目录（`transforms/` 或 `actions/`）下创建目录 |
| 2 | 在父 `pom.xml` 中添加 `<module>` |
| 3 | 将 parent 设置为聚合 POM（如 `hop-plugins-transforms`） |
| 4 | 在所有文件中包含 ASF 许可证头 |
| 5 | 提交前运行 `./mvnw spotless:apply` |

## XML配置示例

### assembly.xml 配置示例

```xml
<?xml version="1.0" encoding="UTF-8"?>
<assembly xmlns="http://maven.apache.org/plugins/maven-assembly-plugin/assembly/1.1.3"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/plugins/maven-assembly-plugin/assembly/1.1.3 
          http://maven.apache.org/xsd/assembly-1.1.3.xsd">
    
    <id>dist</id>
    <formats>
        <format>zip</format>
    </formats>
    
    <fileSets>
        <fileSet>
            <directory>${project.build.directory}</directory>
            <outputDirectory></outputDirectory>
            <includes>
                <include>*.jar</include>
            </includes>
        </fileSet>
        <fileSet>
            <directory>${project.build.directory}/lib</directory>
            <outputDirectory>lib</outputDirectory>
            <includes>
                <include>*.jar</include>
            </includes>
        </fileSet>
    </fileSets>
</assembly>
```

### 插件 pom.xml 配置示例

```xml
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    
    <groupId>org.apache.hop</groupId>
    <artifactId>hop-transform-sample</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <packaging>jar</packaging>
    
    <name>hop-transform-sample</name>
    <description>Sample transform plugin for Hop</description>
    
    <parent>
        <groupId>org.apache.hop</groupId>
        <artifactId>hop-plugins-transforms</artifactId>
        <version>1.0.0-SNAPSHOT</version>
        <relativePath>../pom.xml</relativePath>
    </parent>
    
    <properties>
        <jandex.version>3.1.6</jandex.version>
    </properties>
    
    <dependencies>
        <dependency>
            <groupId>org.apache.hop</groupId>
            <artifactId>hop-core</artifactId>
        </dependency>
        <dependency>
            <groupId>org.apache.hop</groupId>
            <artifactId>hop-engine</artifactId>
        </dependency>
        <dependency>
            <groupId>org.apache.hop</groupId>
            <artifactId>hop-ui-swt</artifactId>
            <scope>provided</scope>
        </dependency>
    </dependencies>
    
    <build>
        <plugins>
            <plugin>
                <groupId>io.smallrye</groupId>
                <artifactId>jandex-maven-plugin</artifactId>
                <version>${jandex.version}</version>
                <executions>
                    <execution>
                        <id>make-index</id>
                        <goals><goal>jandex</goal></goals>
                    </execution>
                </executions>
            </plugin>
        </plugins>
    </build>
</project>
```

## 注意事项

- 插件必须包含 ASF 许可证头
- Jandex 索引是插件加载的必要条件
- 避免将已存在于 hop/lib 中的 jar 包重复打包到插件 zip 中
- 插件 ID 在其类型范围内必须唯一
- 建议参考现有插件的结构和代码风格
