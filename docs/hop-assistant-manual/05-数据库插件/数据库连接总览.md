# 数据库插件

Qi 数据治理平台 为许多数据库类型提供了优化支持。如果您使用的是不支持的数据库，您始终可以创建一个通用连接（Generic connection）。

## 数据库插件概述

Qi 数据治理平台 开箱即用地支持数十种数据库。如果您偏好的数据库没有专门的支持，您通常仍然可以通过通用数据库连接进行连接。

## 如何创建数据库连接

要创建数据库连接，请转到元数据视图（metadata perspective），右键单击 `Relational Database Connections`（关系型数据库连接），然后选择 `New`（新建）。

该连接会保存在中心位置，随后可供所有管道（pipeline）和工作流（workflow）使用。

如果您已将项目设置为与 Hop 配合工作，则数据库信息将位于 `{PROJECT_HOME}/metadata/rdbms` 文件夹中。对于每个连接，此文件夹中会生成一个独立的 .json 文件。此 json 文件将以连接名称命名，并包含所有连接信息。

## 添加 JDBC 驱动

Qi 数据治理平台 为所有在兼容 Apache 公共许可证的许可证下提供其驱动的数据库包含了 JDBC 驱动程序。Qi 数据治理平台 附带的所有 JDBC 驱动程序都可以在您的 Qi 数据治理平台 安装文件夹的 `lib/jdbc` 中找到。

以下各小节中各种数据库类型的文档将向您展示是否包含您所用数据库的驱动。如果包含，连接编辑器还将显示已安装驱动程序的版本。如果不包含该驱动程序，文档将引导您找到下载位置。

## HOP_SHARED_JDBC_FOLDERS 环境变量

将 `HOP_SHARED_JDBC_FOLDERS` 环境变量设置为包含您附加 JDBC 文件夹的文件夹。将它们放在一个中心文件夹中有助于您轻松地升级或更改 Qi 数据治理平台 安装，而无需每次都添加 JDBC 驱动程序。

此变量接受一个以逗号分隔的列表，以指向多个目录；未设置时的默认值为 `lib/jdbc`，例如：`<PATH_TO_YOUR_HOP_INSTALLATION>/lib/jdbc,<PATH_TO_YOUR_JDBC_FOLDER>`。

为避免冲突，请确保每个数据库只有一个驱动程序，并确保您没有多个驱动程序副本，例如同时存在于 `HOP_SHARED_JDBC_FOLDERS` 和 `hop/lib/jdbc` 文件夹中。

## Generic（通用）连接

当您要使用的数据库类型尚无特定支持时，您可以使用通用连接。

要使用通用连接，您必须将您的 jdbc 驱动程序复制到 `<PATH_TO_YOUR_HOP_INSTALLATION>/lib/jdbc` 文件夹或您的 `HOP_SHARED_JDBC_FOLDERS` 文件夹中。

请查阅您数据库的驱动程序类和 URL 语法的文档以创建连接。

在 Driver Class（驱动程序类）字段中指定您的驱动程序类，例如，如果您使用 PostgreSQL，则该类为 `org.postgresql.Driver`。

## 高级属性

Advanced（高级）选项卡允许您为数据库连接指定许多附加属性。

| 属性 | 描述 |
|------|------|
| 支持 Boolean 数据类型（Supports the Boolean data type） | 是否支持 Boolean？ |
| 支持 Timestamp 数据类型（Supports the Timestamp data type） | 是否支持 Timestamp？ |
| 为数据库中的所有标识符加引号（Quote all identifiers in database） | 为生成的 SQL 语句在所有标识符周围添加引号。 |
| 强制所有标识符为小写（Force all identifiers to lower case） | 将生成的 SQL 语句的所有标识符更改为小写。 |
| 强制所有标识符为大写（Force all identifiers to upper case） | 将生成的 SQL 语句的所有标识符更改为大写。 |
| 保留保留字的大小写（Preserve case of reserved words） | 不要更改生成的 SQL 语句中所有保留字的大小写。 |
| 首选架构名称（The preferred schema name） | 默认使用的架构名称（可被覆盖）。 |
| 连接后要运行的 SQL 语句（; 分隔）（The SQL statements to run after connection (; separated)） | 以分号（';'）分隔的 SQL 语句列表，这些语句需要在创建连接后执行。 |

## 选项

Options（选项）表包含可添加到您的 JDBC 驱动程序的键/值对列表。请查阅您的数据库 JDBC 驱动程序文档以获取正确的语法。

例如，要向 MS SQL 数据库添加附加选项，您可以添加选项以实现如下所示的 JDBC URL。Qi 数据治理平台 会在后台处理这些属性，通常无需手动修改 JDBC URL（但您*可以*修改，而在 `Generic` 连接中，您*必须*修改）。

```
jdbc:sqlserver://localhost:1433;" +
     "databaseName=AdventureWorks;integratedSecurity=true;" +
     "encrypt=true;trustServerCertificate=true
```

| 键 | 值 |
|----|----|
| integratedSecurity | true |
| encrypt | true |
| trustServerCertificate | true |
