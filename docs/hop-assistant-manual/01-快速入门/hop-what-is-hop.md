# 什么是 Qi Hop?

## 可视化设计与元数据

Qi Hop 是 **H**op **O**rchestration **P**latform 的缩写，是一个数据编排和数据工程平台，旨在简化数据和元数据编排的各个方面。
Hop 让你专注于要解决的问题，而不受技术干扰。
简单任务应该容易，复杂任务应该可行。

Hop 允许数据专业人员以**可视化**的方式工作，使用**元数据**来描述数据应如何处理。
可视化设计使数据开发者能够专注于做*什么*，而非*如何*完成。
这种对任务本身的关注使 Hop 开发者比编写代码时更加高效。

## 架构

Hop 的设计尽可能灵活：核心是一个小而强大的 Hop 引擎。
所有功能都通过 plugin 添加：默认的 Hop 安装包含约 400 个 plugin。
你可以根据需要移除或添加第三方 plugin，将 Hop 定制为你所需的样子。
Hop 专为在各种场景下运行而设计，从物联网到海量数据，从本地到云端，从裸机操作系统到容器和 Kubernetes。

## 灵活的运行时

Hop 开发者在一个名为 Hop GUI 的可视化开发环境中创建 **Workflow** 和 **Pipeline**。
这些 Workflow 和 Pipeline 可以在多种引擎上执行：Workflow 和 Pipeline 可以在原生 Hop 引擎上运行，支持本地和远程运行。
Pipeline 还可以通过 Apache Beam 运行时配置在 Apache Spark、Apache Flink 和 Google Dataflow 上运行。
在 Workflow 和 Pipeline 中，可以对数据应用数百种操作：从各种源平台读取和写入，以及合并、丰富、清洗和以多种其他方式处理数据。
根据引擎和所选功能，你的数据可以以批处理、流式或批处理/流式混合方式处理。

## 使用场景

Hop 的一些常见使用场景包括：

- 利用云、集群和大规模并行处理环境，将大型数据集加载到数据库中。
- 数据仓库填充，内置支持缓慢变化维度（SCD）、变更数据捕获（CDC）和代理键创建。
- 在不同的数据架构之间进行集成，整合关系型数据库、文件、NoSQL 数据库（如 Neo4j、MongoDB、Cassandra 等）。
- 不同数据库和应用程序之间的数据迁移。
- 数据探查和数据清洗。
