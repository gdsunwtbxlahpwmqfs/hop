# 日志插件架构（Logging Plugins Architecture）

## 简介

Qi 数据治理平台 提供了对日志插件的支持。

日志插件架构允许开发者将不同的日志框架（例如 log4j 等）集成到 Hop 的日志系统中。通过插件机制，您可以根据组织的日志管理需求，自定义日志的输出格式、输出位置以及日志级别处理方式。

## 架构说明

Hop 的日志系统是可扩展的。默认情况下，Hop 使用其内置的日志实现。如果需要使用其他日志框架（如 log4j），可以通过实现日志插件接口来集成。

日志插件示例代码（例如 log4j 插件）可作为参考实现。

## 适用场景

- 需要将 Hop 日志集成到企业统一日志管理系统中
- 需要使用特定的日志格式或日志输出目标
- 需要对日志进行更精细的级别控制和路由
