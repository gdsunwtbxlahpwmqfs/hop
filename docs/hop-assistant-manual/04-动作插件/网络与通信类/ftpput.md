# 通过 FTP 上传文件

## 描述

Put A File With FTP action 将文件上传到 FTP 服务器。

## 选项

### 常规选项卡

| 选项 | 描述 |
|---|---|
| Action 名称 | workflow action 的名称。 |
| FTP 服务器名称/IP 地址 | 服务器名称或 IP 地址。 |
| 端口 | FTP 服务器连接的端口号。 |
| 用户名 | 与 FTP 服务器账户关联的用户名。 |
| 密码 | 与 FTP 服务器账户关联的密码。 |
| 代理主机 | 代理服务器主机名。 |
| 代理端口 | 代理服务器端口号。 |
| 代理用户名 | 代理服务器账户用户名。 |
| 代理密码 | 代理服务器账户密码。 |
| 测试连接 | 测试已配置的连接。 |
| 二进制模式 | 如果文件必须以二进制模式传输，请启用。 |
| 超时 | FTP 服务器超时时间（秒）。 |
| 使用主动 FTP 连接 | 如果使用主动模式连接 FTP 服务器，请启用；您必须设置防火墙以接受到 FTP 客户端将打开的端口的连接。 |
| 控制编码 | 选择用于确定文件名中特殊字符的字符编码。 |

### Files 选项卡

| 选项 | 描述 |
|---|---|
| 本地目录 | 要从中上传的本地目录。 |
| 浏览... | 浏览文件系统以选择本地目录。 |
| 通配符（正则表达式） | 如果要上传多个文件，请指定定义要上传文件集的正则表达式。 |
| 传输后删除文件？ | 启用以在文件上传后从本地目录中删除。 |
| 不覆盖文件 | 启用以跳过远程目录中已存在的文件。 |
| 远程目录 | FTP 服务器上要上传文件到的目录。 |
| 测试文件夹 | 测试远程目录是否存在。这将连接到 FTP 服务器并尝试访问指定目录。如果此目录不存在，上传将失败。 |

### Socks 代理

| 选项 | 描述 |
|---|---|
| 主机 | Socks 代理服务器的主机名。 |
| 端口 | Socks 代理连接的端口号。 |
| 用户名 | 与 Socks 代理服务器账户关联的用户名。 |
| 密码 | 与 Socks 代理服务器账户关联的密码。 |
