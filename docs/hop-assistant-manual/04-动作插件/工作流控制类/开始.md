# 开始（Start）

`Start` 动作是一个特殊的动作，它是所有工作流执行的起点。

> 注意：每个工作流必须且只能有一个开始动作。

开始动作具有重复工作流的附加选项。该功能在本地和远程工作流引擎中均可使用，但主要为历史原因而保留，不应将其视为 `cron`、Apache Airflow 或任何其他调度方式的替代方案。

当您激活重复选项时，工作流将根据您指定的间隔持续运行并重新启动。可以是固定日期的特定时间戳，也可以是每隔 x 秒/分钟。

虽然这不能替代调度器，但可用于基本调度。请注意，使用此功能会使工作流保持运行状态，这意味着 Java 进程将继续存在并占用系统资源。

## 主要选项

| 选项 | 说明 |
|------|------|
| 动作名称（Action name） | 工作流动作的名称，在单个工作流中必须唯一。 |
