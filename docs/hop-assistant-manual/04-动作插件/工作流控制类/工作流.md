# 工作流（Workflow）

`Workflow` 动作将先前定义的工作流作为当前工作流的子工作流来执行。

为方便使用，也可以在对话框中创建新工作流，点击"新建工作流"按钮即可。

使用工作流动作来执行先前定义的工作流。

这允许您执行"功能分解"。也就是说，您可以使用它们将工作流分解为更易于管理的单元。

例如，您不应该使用包含 500 个动作的一个工作流来编写数据仓库加载。

最好创建较小的工作流并将它们组合起来。

## 另请参阅

* 管道动作（Pipeline action）：从工作流执行管道。
* 工作流执行器转换（Workflow Executor transform）：从管道执行工作流。
* 管道执行器转换（Pipeline Executor transform）：从管道执行子管道。
