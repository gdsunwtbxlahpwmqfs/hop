# 等待并行分支（Join Action）

## 功能概述


*Join* 动作用于同步工作流中的并行分支。它作为一个控制点，*等待同一级别的所有前序并行分支*完成后再继续执行下一个动作。
这消除了仅为在并行任务完成后汇合并继续执行而创建单独子工作流的需要。

## 使用说明

传统上，在 Qi 数据治理平台 中，如果您想并行运行工作流的某些部分，然后继续顺序流程，您需要将这些并行部分隔离在子工作流中。这是为了防止下游动作过早执行。

有了新的 *Join* 动作，就不再需要这种变通方法了。

使用 Join 动作的步骤：

1. 配置工作流，使单个动作具有多个输出跳（hops）。
2. 在要并行运行的输出跳上启用*并行执行*。
3. 在所有并行分支汇聚的节点添加 *Join* 动作。
4. 使用常规（非并行）跳将所有并行分支连接到 *Join* 动作。
