# 文件元数据（File Metadata）

## 功能概述


文件元数据转换通过扫描文件来确定其元数据结构或布局。
当您需要读取结构化文本文件（例如 CSV、TSV），但事先不知道确切布局时，请使用此转换。
此文件中提供的信息可用于稍后实际读取文件，例如通过元数据注入（Metadata Injection）。
检测到的布局取决于扫描的行数。例如，如果扫描了文件的前 100 行且第一个字段被检测为整数，则该字段在后续行中可能包含字母数字字符。将"限制扫描行数（limit scanned rows）"设置为 0 可以确保扫描整个文件并正确检测布局，尽管这对于大文件可能耗时甚至不可行。

## 主要选项

| 选项 | 说明 |
|------|------|
| 文件名字段（Filename field） | 包含要扫描的文件名的输入字段 |
| 限制扫描行数（Limit scanned rows） | 限制扫描的行数，设为 0 表示扫描整个文件 |
| 输出字段（Output fields） | 检测到的元数据信息输出字段 |

## 注意事项

- 支持 Qi 数据治理平台 Engine 引擎；Spark、Flink、Dataflow 引擎支持情况视具体实现而定。
- 检测到的布局信息可通过元数据注入传递给后续读取文件的转换。
