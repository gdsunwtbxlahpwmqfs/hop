# Beam 窗口（Beam Window）

Beam 窗口转换使用 Beam 执行引擎添加基于事件时间的窗口函数。该转换专为在 Beam 引擎上运行的管道设计，用于将无界数据流划分为有限的、可处理的批次。

## 支持的引擎

| 引擎 | 是否支持 |
|------|---------|
| Hop Engine | ❌ 不支持 |
| Spark | ✅ 支持 |
| Flink | ✅ 支持 |
| Dataflow | ✅ 支持 |

## 主要选项

| 选项 | 说明 |
|------|------|
| 转换名称（Transform name） | 转换的名称，在单个管道中必须唯一。 |
| 窗口类型（Window type） | 可选：固定（Fixed）、滑动（Sliding）、会话（Session）、全局（Global）。 |
| 窗口大小（持续时间，秒）（Window size） | 设置窗口持续时间大小（秒），默认 60。 |
| 每 x 秒（滑动窗口）（Every x seconds） | 设置滑动窗口的持续时间（秒）。 |
| 窗口起始字段（Window start field） | 包含窗口开始时间的字段。 |
| 窗口结束字段（Window end field） | 包含窗口结束时间的字段。 |
| 窗口最大字段（Window max field） | 包含事件之间最大持续时间的字段。 |

## 窗口类型说明

### 固定窗口（Fixed）
固定或翻滚（tumbling）窗口用于将数据重复切分为不同的时间段，各窗口之间不重叠。一个事件不能属于多个窗口。

### 滑动窗口（Sliding）
滑动窗口仅在事件发生时产生输出，并持续向前移动。每个窗口至少包含一个事件，并且可以重叠。一个事件可以属于多个窗口。

### 会话窗口（Session）
会话窗口将相近时间到达的事件分组，并过滤掉没有数据的时段。窗口在第一个事件发生时开始，并扩展以包含在指定超时时间内的新事件。如果事件持续发生，窗口会一直扩展，直到达到最大持续时间。

### 全局窗口（Global）
全局窗口将所有事件分配到同一个窗口中。
