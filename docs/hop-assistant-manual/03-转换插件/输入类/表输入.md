# 表输入

表输入（Table Input）转换是 Apache Hop 中最常用的转换之一。它通过数据库连接和 SQL 查询从关系型数据库中读取数据，并将数据引入管道。

该转换支持以下参数化方式：

- **Apache Hop 变量**：例如 `${customerId}`
- **来自先前转换的输入字段**：使用 `?` 占位符的 JDBC 预编译语句

这些特性使表输入非常适合用于动态且可复用的数据检索逻辑。

## 主要选项

| 选项 | 说明 |
|------|------|
| 转换名称 | 转换的名称 |
| 数据库连接 | 用于查询的数据库连接 |
| SQL 查询 | 用于检索数据的 SQL 语句，可包含变量和 `?` 占位符 |
| 允许延迟转换 | 是否允许惰性转换以提升性能 |
| 替换变量 | 是否在 SQL 中替换 Hop 变量 |
| 从步骤插入数据 | 选择将先前步骤的数据插入到 SQL 的 `?` 占位符中 |

## 注意事项

- 该转换支持 Hop Engine、Spark、Flink、Dataflow 等多种引擎。
- 使用 `?` 占位符时，可通过"从步骤插入数据"选项将先前转换的字段值绑定到 SQL 查询中（JDBC 预编译语句）。
- 变量替换与预编译语句可以结合使用，实现灵活的动态查询。
