# 组合查找/更新

## 功能概述


组合查找/更新（Combination Lookup/Update）转换允许你将信息存储在杂项维度（junk-dimension）表中。该转换也可用于维护 Kimball 纯类型 1 维度。
该转换会执行以下操作：
1. 在维度表中从输入流查找业务键字段 field1...fieldn 的组合；
2. 如果该业务键字段组合已存在，返回其技术键（代理 ID）；
3. 如果该业务键组合尚不存在，则插入带有新键字段的行并返回其（新的）技术键；
4. 将所有输入字段（包括返回的技术键）放到输出流上，但如果"移除查找字段"为 true，则移除所有业务键字段。
该转换从具有业务键的数据中创建/维护技术键。经过此转换后，维度表的其余数据更改都可以作为更新执行，因为业务键的行要么已存在，要么已被创建。
该转换仅维护键信息。

## 注意事项

- 该转换用于维护杂项维度表，自动管理技术键（代理键）。
- 业务键字段组合唯一确定一条维度记录。
- 适用于维度建模中的杂项维度管理。
