# 文本文件输出

## 功能概述


文本文件输出（Text File Output）转换用于将数据导出为文本文件格式。
这通常用于生成可被电子表格应用程序读取的逗号分隔值（CSV 文件）。也可以通过在"字段"选项卡中设置字段长度来生成固定宽度文件。
你可以选择使用"架构定义（Schema Definition）"或手动定义所需字段的布局。如果决定使用架构定义来定义字段布局，可使用架构映射（Schema Mapping）转换根据所选架构定义调整传入流。
`ignore manual fields`（忽略手动字段）选项会忽略在转换字段布局中手动定义的所有字段，仅使用架构定义中指定的布局。

## 主要选项

| 选项 | 说明 |
|------|------|
| 转换名称 | 转换的名称 |
| 文件名 | 输出文件名 |
| 扩展名 | 输出文件扩展名 |
| 分隔符 | 字段之间的分隔符 |
| 封闭符 | 文本字段的封闭符 |
| 格式 | 输出格式（DOS 或 Unix） |
| 编码 | 输出文件编码 |
| 追加 | 是否追加到已有文件 |
| 包含表头 | 是否在文件头部写入列名 |

## 注意事项

- 该转换支持 Qi 数据治理平台 Engine、Spark、Flink、Dataflow 等多种引擎。
- 通过在字段选项卡中设置长度，可生成固定宽度文件。
- 支持使用架构定义来规范输出字段布局。
