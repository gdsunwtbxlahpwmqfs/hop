# 分析查询

分析查询（Analytic Query）转换允许你在管道中向前和向后窥探行。

常见用例示例：

- 通过按订单日期排序行，并向后 LAG 一行获取上一个订单时间来计算"订单间隔时间"。
- 通过向前 LEAD 一行并确定用户在此页面上停留了多少秒来计算网页浏览的"持续时间"。

## 主要选项

| 选项 | 说明 |
|------|------|
| 转换名称 | 转换的名称 |
| 分组字段 | 用于分组的字段 |
| 向前窥探行数（LEAD） | 向前查看的行数 |
| 向后窥探行数（LAG） | 向后查看的行数 |
| 字段 | 要提取的前后行字段 |

## 注意事项

- 该转换仅支持 Hop Engine，不支持 Spark、Flink、Dataflow。
- 类似 SQL 窗口函数中的 LEAD 和 LAG 功能。
- 适用于计算时间间隔、移动平均等需要访问相邻行的分析场景。
