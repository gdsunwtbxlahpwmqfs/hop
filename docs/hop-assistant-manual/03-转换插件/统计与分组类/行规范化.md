# 行规范化

行规范化（Row Normaliser）转换将输入流的列转换为行。

你可以使用此转换来规范化重复的列组。

**重要**：当将具有不同元类型（如字符串和整数）的多个列合并到新字段时，不会执行自动类型转换，而是设置第一个元类型。这种转换的缺失可能导致后续对结果数据行的转换出现问题。强烈建议在规范化之前确保放入同一字段的数据类型对齐。

## 主要选项

| 选项 | 说明 |
|------|------|
| 转换名称 | 转换的名称 |
| 类型字段 | 输出中标识值来源列的字段名 |
| 值字段 | 输出中包含值的字段名 |
| 规范化字段 | 列到行的映射定义（列名、类型值等） |

## 注意事项

- 该转换支持 Hop Engine、Spark、Flink、Dataflow 等多种引擎。
- 将宽表（多列）转换为长表（多行），是行反规范化的逆操作。
- 合并不同数据类型的列时不会自动转换类型，需提前确保类型对齐。
