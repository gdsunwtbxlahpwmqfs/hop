# 中止

中止（Abort）转换在接收到输入数据后立即中止正在运行的管道。该转换的主要用例是在发生意外或不需要的情况时抛出错误。

例如，你可以使用此转换，使管道在超过 x 行数据流经错误跳后被中止。

## 主要选项

| 选项 | 说明 |
|------|------|
| 转换名称 | 转换的名称 |
| 中止阈值 | 接收到多少行后中止管道 |
| 中止后管道日志消息 | 中止管道时记录的消息 |

## 注意事项

- 该转换支持 Hop Engine、Spark、Flink、Dataflow 等多种引擎。
- 通常与错误处理跳配合使用，在累积一定数量的错误行后中止管道。
- 中止会抛出错误并停止管道执行。
