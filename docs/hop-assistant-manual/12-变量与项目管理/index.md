# 项目与环境

## 概念与架构

### 项目

大多数开发者通常同时处理多个项目和模块。
例如，当一个项目成熟并进入用户验收测试阶段时，新项目的开发工作可能已经开始了。

不同的 workflow 和 pipeline 集合需要至少为开发、验收和生产环境进行管理。
每个项目和每个环境都有自己的一套变量和数据库、文件路径等配置。

Hop 希望让 Hop 开发者能够轻松地同时使用一个或多个项目。

项目是你在数据编排解决方案中所需的所有文件的集合。
通常包括 metadata、pipeline、workflow、参考文件、文档等。
为了符合标准开发最佳实践，你应该将所有这些文件提交到版本控制系统（如 git）中。

> **💡 提示:** 项目变量只应在需要项目级别的变量时使用。所有连接基础设施的变量，例如数据库连接参数、邮件服务器等在不同环境中取不同值的变量，应在环境级别创建。

> **💡 提示:** 项目配置存储在 hop-config.json 中，默认从 `hop/config` 读取。使用 `HOP_CONFIG_FOLDER` 操作系统变量将你的 Hop 配置存储在 Hop 文件夹之外的文件夹中。这样当你切换 Hop 安装或升级到更新的 Hop 版本时，可以保留你的项目列表。

项目可以从父项目继承 metadata 和变量。

Hop GUI 中的项目会记住已打开的文件、缩放级别和其他 UI 设置。

### 环境

通常不包含在项目文件集中的是使项目在给定环境中正确运行所需的 metadata 配置和变量值。

例如，在开发笔记本电脑上你可能想将变量 `{openvar}DB_HOSTNAME{closevar}` 设置为 `localhost`，而在生产服务器上此值可能指向生产数据库服务器。
这就是 Hop 环境的用武之地。

环境是项目生命周期环境的简称。
它描述了项目在其生命周期中从开发到测试到验收再到生产所处的阶段。
它也可以描述持续集成环境中的项目等。
因此，以下属性定义了一个环境：

- 其名称
- 项目名称
- 阶段
- 你想要用来定义环境特定变量的配置文件

与项目一样，环境也是在一个或多个配置文件中定义的。

> **💡 提示:** 将你的环境配置文件存储在项目文件夹之外。
你甚至可能希望将它们提交到单独的版本控制仓库中。
