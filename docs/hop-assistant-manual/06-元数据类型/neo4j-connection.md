# Neo4j 连接

## 描述

连接可以通过 logo 创建和编辑（点击 logo 进行编辑），也可以通过下拉菜单（向下箭头）创建，该菜单位于各种 Neo4j 对话框中每行连接的右侧。

也可以在 Hop GUI 的 metadata perspective 中管理。

Neo4j 连接以 JSON 格式序列化存储在 Hop 元数据文件夹中：

`metadata/neo4j-connection/`

## 用户界面

在大多数情况下，Neo4j [驱动程序](https://neo4j.com/docs/java-manual/current/get-started/)可以自动确定协议和安全选项，让你可以专注于最基本的选项：服务器、端口、用户名和密码。
与往常一样，我们建议你永远不要硬编码任何设置，而是使用可以在环境配置文件中定义的环境变量。这样，同一个 Neo4j 连接可以在多个环境中使用，而无需更改 Pipeline 或 Workflow。

## 高级选项

### 连接活性检查超时（ms）

在池中空闲时间超过此超时的池化连接将在再次使用前进行测试，以确保它们仍然活跃。
如果此选项设置得太低，获取连接时将产生额外的网络调用，从而导致性能下降。
如果设置得太高，你可能会收到由不再活跃的连接支持的会话，这将导致应用程序中出现异常。
假设数据库正在运行，如果你重试获取会话，这些异常将消失。
因此，此参数调整了应用程序出现连接问题的可能性与性能之间的平衡。

通常你不需要调整此参数。
默认不进行连接活性检查。
值 0 表示连接将始终进行有效性测试，负值表示连接永远不会被测试。

### 最大连接生命周期（ms）

超过此阈值的池化连接将被关闭并从池中移除。
这种丢弃在获取连接时发生，因此新会话永远不会由旧连接支持。
将此选项设置为较低的值会导致高连接流失率，并可能导致性能下降。

建议将最大生命周期设置为略小于网络设备中配置的值（负载均衡器、代理、防火墙等也可能限制最大连接生命周期）。

此设置也可以与连接活性检查超时（ms）值结合使用。
在这种情况下，建议将活性检查设置为小于网络设备的值，并将最大生命周期设置为合理较大的值以定期"续订"连接。

默认最大连接生命周期为 1 小时。
零和负值表示不检查生命周期。

### 最大连接池大小

配置到单个数据库的连接池中的最大连接数。
当与直接驱动程序一起使用时（为使用 'bolt' 方案的 URI 创建），此设置限制池中的总连接数。
当与路由驱动程序一起使用时（为使用 'neo4j' 方案的 URI 创建），它将限制每个集群成员的连接数。

当达到限制时，将在最多配置的连接获取超时（Connection Acquisition Timeout(ms)）内尝试获取连接。

默认值为 100。允许使用负值，结果为无限连接池。
不允许使用值 0。

### 连接获取时间（ms）

配置连接获取尝试从连接池获取连接的最长时间。
仅当所有现有连接都在使用中且由于已达到最大连接池大小而无法创建新连接时，此超时才会生效。
当在配置的时间内无法获取连接时，将抛出错误。

默认值为 60 秒。
允许使用负值，结果为无限获取超时。
允许使用值 0，结果为无超时并在连接不可用时立即失败。

### 最大事务重试时间（ms）

指定通过会话读写事务允许事务重试的最长时间。
这些方法将在 ServiceUnavailableException、SessionExpiredException 和 TransientException 上重试给定的工作单元，使用 1 秒的初始延迟进行指数退避。
默认值为 30 秒。

### 手动 URL

你可以指定一系列手动连接 URL，以允许你使用任何你可能想要的高级、遗留、前沿等功能。
有关 Bolt 连接 URI 的更多信息，请参见：https://neo4j.com/docs/driver-manual/current/client-applications/
