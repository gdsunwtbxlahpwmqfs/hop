# 执行信息位置（Execution Information Location）

## 核心功能

执行信息位置决定 Apache Hop 将执行信息发送到何处。

执行完成后，可以从执行信息视图中查阅这些信息。

## 主要参数

| 参数 | 说明 | 默认值 |
| --- | --- | --- |
| Name（名称） | 此执行信息位置使用的名称 | - |
| Description（描述） | 此执行信息位置使用的描述 | - |
| Data logging delay (ms)（数据日志记录延迟，毫秒） | | 2000 |
| Data logging interval (ms)（数据日志记录间隔，毫秒） | | 5000 |
| Data logging size (rows)（数据日志记录大小，行数） | | 1000 |
| Location type（位置类型） | 要写入的位置插件类型。根据类型不同，可输入更多特定于插件的选项 | |

## 位置类型特定选项

### 文件位置（File location）

通过文件位置，您可以将执行信息写入本地文件系统某个路径下的文件中。

选项：
- **Root folder（根文件夹）**：写入执行信息的文件夹。

### 缓存文件位置（Caching File Location）

缓存文件位置提供基于文件的缓存能力。
