# 工作流日志（Workflow Log）

## 核心功能

允许使用一个管道来记录某个工作流的活动。

工作流日志将运行中工作流的日志信息流式传输到一个管道。接收管道随后可以处理这些日志信息，例如写入关系型或 NoSQL 数据库、Kafka 主题等。

接收管道的唯一要求是它必须以"工作流日志（Workflow Logging）"转换作为起始转换。除此之外，该日志管道就是"一个普通的管道"。

## 示例

Hop 的示例项目附带了一个工作流日志示例。可在元数据视图中查看名为 `workflow-log-example` 的工作流日志。该工作流日志配置为将工作流 `${PROJECT_HOME}/reflection/generate-fake-books.hwf` 的日志信息发送到日志管道 `${PROJECT_HOME}/reflection/workflow-log-example.hpl`。
