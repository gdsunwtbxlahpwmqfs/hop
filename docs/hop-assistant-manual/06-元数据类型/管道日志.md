# 管道日志（Pipeline Log）

## 核心功能

允许使用另一个管道来记录某个管道的活动。

管道日志将运行中管道的日志信息流式传输到另一个管道。接收管道随后可以处理这些日志信息，例如写入关系型或 NoSQL 数据库、Kafka 主题等。

接收管道的唯一要求是它必须以"管道日志（Pipeline Logging）"转换作为起始转换。除此之外，该日志管道就是"一个普通的管道"。

## 示例

Hop 的示例项目附带了一个管道日志示例。可在元数据视图中查看名为 `pipeline-log-example` 的管道日志。该管道日志配置为将管道 `${PROJECT_HOME}/reflection/generate-fake-books.hpl` 的日志信息发送到日志管道 `${PROJECT_HOME}/reflection/pipeline-log-example.hpl`。
