# 管道变量解析器（Pipeline Variable Resolver）

## 核心功能

我们可以创建一个类型为"管道变量解析器"的新变量解析器元数据元素。

所使用的选项非常简单：

- **要使用的管道文件名**：用于运行以解析变量表达式的管道文件名。
- **管道运行配置名称**：必须引用类型为"原生本地（Native Local）"的运行配置。
- **管道中包含要解析表达式的变量名称**。
- **用于获取字段值的输出字段名称**。

这种设置为您提供了极大的灵活性。也许您每个数据库、每个系统都有一个配置文件……在这种情况下，您可以设置变量来包含数据库、系统等的类型或名称，以读取相应的配置文件。您可以从这些替代配置存储中检索多个值，并将它们以单行的形式发送到输出转换。

**重要提示**：仅考虑管道中的第一行输出。

## 变量表达式格式

变量表达式（一如既往）采用以下格式：

```
#{name:expression:value-key}
```

- name：要使用的变量解析器元数据元素的名称
- expression：要解析的表达式
- value-key：要选取的输出字段的名称
