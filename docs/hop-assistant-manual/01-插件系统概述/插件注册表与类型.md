# 插件注册表与类型

## 功能概述


Qi 数据治理平台是一个高度可扩展的数据/元数据编排平台。其几乎全部功能均以"插件"形式实现，开发者可通过插件在 20 余个不同领域扩展 Qi 数据治理平台的能力。
Qi 数据治理平台维护着一个中心化的插件注册表（Plugin Registry），用于管理所有可用的 Qi 数据治理平台插件。插件按插件类型进行组织。注册表同时跟踪为各插件创建和使用的类加载器（ClassLoader），以避免重复加载。

## 插件注册表（Plugin Registry）

### 获取插件注册表实例

在任意位置均可通过以下方式获取插件注册表实例：

```java
PluginRegistry.getInstance()
```

### 插件发现机制

插件注册表通过以下两种方式查找和加载插件：

1. **读取 XML 文件**：当待加载的插件类位于 classpath 中时使用此方式。该技术用于核心插件和部分 GUI 插件。相关文件可在 `core/src/resources`、`engine/src/resources` 和 `ui/src/resources` 中找到。文件位置通过 `getXmlPluginFile()` 方法定义。

2. **扫描 `plugins/` 子文件夹**：扫描 jar 文件，查找带有插件类型所指定的注解和接口的类，随后加载这些类并将其添加到插件注册表中。

## 插件类型（Plugin Types）

插件允许你在超过 20 个不同领域扩展 Qi 数据治理平台的功能。每个插件类型由一个实现了 `IPluginType<T extends Annotation>` 接口的类来表示。为方便起见，你可以继承 `BasePluginType<T extends Annotation>` 类。

## 插件信息字段

可以通过插件注册表使用 ID、类名搜索插件，也可列出所有插件和类型。插件包含大量信息并实现 `IPlugin` 接口。

| 字段 | 类型 | 是否可选 | 说明 |
|------|------|----------|------|
| `ids` | String[] | 必填 | 插件在其类型范围内的唯一 ID。一个插件可以有多个 ID，以便将多个插件合并为一个并保持向后兼容。 |
| `name` | String | 可选 | 用户界面中显示的插件名称 |
| `description` | String | 可选 | 用户界面中显示的插件描述 |
| `category` | String | 可选 | 用于在 UI 中组织某些类型的插件（如转换和动作） |
| `imageFile` | String | 可选 | 代表该插件图标的 SVG 文件路径 |
| `documentationUrl` | String | 可选 | 插件文档的 URL |
| `keywords` | String[] | 可选 | 代表该插件的关键词，便于在 UI 中查找 |
| `nativePlugin` | boolean | 否 | 指示该插件是否可在 classpath（通过 XML 或 API）中找到，还是在插件 classpath 中 |

## Qi 数据治理平台插件的位置

在 Qi 数据治理平台中，所有可选插件位于 `hop/plugins` 文件夹中，该文件夹包含以下插件类别：

| 类别 | 说明 |
|------|------|
| **database（数据库）** | 关系型数据库插件 |
| **actions（动作）** | 工作流动作 |
| **engines（引擎）** | 管道和工作流引擎 |
| **transforms（转换）** | 管道转换 |
| **vfs（虚拟文件系统）** | 虚拟文件系统插件 |
| **misc（杂项）** | 杂项插件。仅当其他插件类型不适用时才选择此类别。 |

## XML配置示例

插件注册表通过 XML 文件配置核心插件，以下是一个典型的插件注册 XML 结构：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<plugin-registry>
  <plugin>
    <id>transform-table-input</id>
    <name>Table Input</name>
    <description>Read data from a database table</description>
    <category>transform</category>
    <imageFile>ui/images/table-input.svg</imageFile>
    <classname>org.apache.hop.pipeline.transforms.tableinput.TableInputMeta</classname>
    <keywords>database,sql,read,input</keywords>
    <nativePlugin>Y</nativePlugin>
  </plugin>
</plugin-registry>
```

## 注意事项

- 插件 ID 在其类型范围内必须唯一
- 每个插件类别对应一个特定的注解和接口
- Jandex 索引用于加速插件注解的搜索
- 插件 jar 文件中必须包含 Jandex 索引才能被正确加载
