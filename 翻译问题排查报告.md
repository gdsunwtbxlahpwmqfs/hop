# Apache Hop Web UI 中文翻译问题排查报告

## 问题描述

**现象**：在 `hop-config.json` 中设置 `"LocaleDefault": "zh_CN"` 并重启 Web 服务后，菜单栏（File、Edit、View、Tools、Run、Help）仍显示英文。

**环境**：
- 部署方式：Docker（tomcat:10-jdk21）+ docker-compose
- 启动脚本：`dev-scripts/start-web-dev.sh`
- 配置文件：`assemblies/web/target/webapp/config/hop-config.json`

---

## 排查过程

### 阶段一：确认配置读取正确

在 [LanguageChoice.java](file:///Users/yixiao/Documents/a723wlkjgzs_code/mygithub/hop/core/src/main/java/org/apache/hop/i18n/LanguageChoice.java) 构造函数中添加诊断日志，确认：

```
[LanguageChoice] constructor: LocaleDefault from config = 'zh_CN'
[LanguageChoice] resolved defaultLocale = zh_CN
```

**结论**：配置读取正确，`zh_CN` locale 已正确解析。**浏览器语言不影响** —— Hop 的 locale 完全由 `hop-config.json` 决定，与 HTTP 请求头、系统语言无关。

### 阶段二：追踪菜单标签翻译链路

菜单标签的翻译链路：

```
@GuiMenuElement(label = "i18n::HopGui.Menu.File")
  → GuiMenuItem 构造函数
    → BaseGuiElements.getTranslation()
      → BaseMessages.getString(packageName, key, resourceClass)
        → GlobalMessages.calculateString()
          → GlobalMessageUtil.calculateString()
            → ResourceBundle.getBundle() with GlobalMessageControl
              → GlobalMessageControl.newBundle()
                → new PropertyResourceBundle(new InputStreamReader(stream, UTF_8))
```

**关键点**：菜单标签在 `GuiMenuItem` 构造时翻译**一次**，之后缓存在 `GuiRegistry` 单例中。

### 阶段三：定位 Resource Bundle 加载失败

在 [GlobalMessageUtil.java](file:///Users/yixiao/Documents/a723wlkjgzs_code/mygithub/hop/core/src/main/java/org/apache/hop/i18n/GlobalMessageUtil.java) 的 `newBundle()` 方法中添加诊断日志，发现：

```
[GMU-newBundle] EXCEPTION creating bundle for
  org/apache/hop/ui/hopgui/messages/messages_zh_CN.properties :
  java.lang.IllegalArgumentException : Malformed \uxxxx encoding.
```

**中文 properties 文件加载抛出异常**，导致 `ResourceBundle.getBundle()` 回退到英文 `messages_en_US.properties`。

### 阶段四：定位畸形 Unicode 转义序列

Java `.properties` 文件要求 `\u` 后**必须恰好 4 位十六进制数字**。使用 Python 脚本扫描文件：

```python
re.finditer(r'\\u([0-9a-fA-F]*)', line)
```

发现 [messages_zh_CN.properties](file:///Users/yixiao/Documents/a723wlkjgzs_code/mygithub/hop/ui/src/main/resources/org/apache/hop/ui/hopgui/messages/messages_zh_CN.properties) 中有 **6 处** 畸形序列：

| 行号 | 错误值 | 正确值 | 对应字符 | 词语 |
|------|--------|--------|----------|------|
| 250 | `\u65E5\uFD7` | `\u65E5\u5FD7` | 日志 | 清除日志 |
| 253 | `\u65E5\uFD7` | `\u65E5\u5FD7` | 日志 | 显示日志 |
| 261 | `\u65E5\uFD7` | `\u65E5\u5FD7` | 日志 | 日志设置 |
| 290 | `\u65E5\uFD7` | `\u65E5\u5FD7` | 日志 | Pipeline 日志 |
| 297 | `\u65E5\uFD7` | `\u65E5\u5FD7` | 日志 | 日志字段 |
| 298 | `\u65E5\uFD7` | `\u65E5\u5FD7` | 日志 | 日志主题 |

**根因**：`\uFD7` 只有 3 位十六进制数字，Java 的 `PropertyResourceBundle`（内部使用 `Properties.load()`）抛出 `IllegalArgumentException: Malformed \uxxxx encoding`，导致**整个 bundle 加载失败**，所有中文翻译全部丢失，回退到英文。

### 阶段五：修复验证

修复后重建，容器日志确认：

```

[GuiMenuItem] id=10000-menu-file | label='文件(&F)'
[GuiMenuItem] id=20000-menu-edit | label='编辑(&E)'
[GuiMenuItem] id=25000-menu-view | label='观察(&V)'
[GuiMenuItem] id=30000-menu-run | label='执行(&R)'
[GuiMenuItem] id=40000-menu-tools | label='工具(&T)'
[GuiMenuItem] id=90000-menu-help | label='帮助(&H)'
```

## 根本原因

**`\u5FD7`（"志"）被错误写成 `\uFD7`（缺少一位 hex 数字 `5`）**。

Java Properties 解析器对 `\u` 转义有严格规则：
- `\u` 后必须紧跟**恰好 4 位**十六进制字符（`[0-9a-fA-F]`）
- 位数不足会抛出 `IllegalArgumentException: Malformed \uxxxx encoding`
- **单个键值对的解析失败会导致整个文件加载失败**（`PropertyResourceBundle` 是原子加载）

这与 `Properties.load()` 在某些实现下的容错行为不同，是排查初期的迷惑点。

## 经验总结

1. **Java `.properties` 文件的 `\u` 转义必须严格 4 位 hex**，缺少任何一位都会导致整个文件加载失败（原子性）
2. **ResourceBundle 回退是静默的**：当目标 locale bundle 加载失败时，会自动回退到候选 locale（en_US），不报错，容易误以为是配置问题
3. **浏览器/系统语言不影响 Hop locale**：Hop 的 i18n 完全由 `LanguageChoice.getDefaultLocale()`（读取 `hop-config.json`）决定，与 HTTP `Accept-Language` 头无关
4. **RAP Web UI 有客户端缓存**：修复后需要用无痕模式或清除浏览器缓存才能看到效果
5. **排查 i18n 问题的有效手段**：在 `ResourceBundle.Control.newBundle()` 添加日志，观察 bundle 加载的成功/失败/异常

## 验证方法

修复后验证步骤：

```bash
# 1. 扫描所有 zh_CN properties 文件中的畸形 \u 序列
python3 -c "
import re, glob
for f in glob.glob('**/messages_zh_CN.properties', recursive=True):
    with open(f, encoding='utf-8') as fh:
        for i, line in enumerate(fh, 1):
            for m in re.finditer(r'\\\\u([0-9a-fA-F]*)', line):
                if len(m.group(1)) != 4:
                    print(f'{f}:{i}: {m.group(0)}')
"

# 2. 重启容器
docker restart hop-web-dev

# 3. 用无痕模式打开 http://localhost:8080 确认菜单为中文
```
