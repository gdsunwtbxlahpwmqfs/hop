开发过程中遇到的典型问题、根因分析与解决方案，便于后续排查同类问题。

## 1. Invalid thread access（访问 /ui 报 500 错误）

### 现象

浏览器访问 `http://localhost:8080/ui`（已登录）时，页面返回：

```
HTTP Status 500 – Internal Server Error
Type: Exception Report
Message: Invalid thread access

org.eclipse.swt.SWTException: Invalid thread access
    org.eclipse.swt.SWT.error(SWT.java:4950)
    org.eclipse.swt.widgets.Display.error(Display.java:1211)
    org.eclipse.swt.widgets.Display.createDisplay(Display.java:962)
    org.eclipse.swt.widgets.Display.create(Display.java:946)
    org.eclipse.swt.graphics.Device.<init>(Device.java:132)
    org.eclipse.swt.widgets.Display.<init>(Display.java:800)
    org.eclipse.swt.widgets.Display.<init>(Display.java:791)
    org.eclipse.rap.rwt.application.AbstractEntryPoint.createUI(AbstractEntryPoint.java:57)
    org.eclipse.rap.rwt.internal.lifecycle.RWTLifeCycle.createUI(RWTLifeCycle.java:177)
    ...
```

### 根因

**桌面版 Cocoa SWT jar 错误地存在于 RAP web 应用的 `WEB-INF/lib` 中。**

```
WEB-INF/lib/
  ├── org.eclipse.swt.cocoa.macosx.aarch64-3.132.0.jar  ← 桌面版 SWT（错误！）
  └── org.eclipse.rap.rwt-4.4.0.jar                      ← RAP 版 SWT（正确）
```

两个 SWT 实现同时存在于 classpath 中。JVM 在某些情况下加载了 Cocoa 版的 `Display` 类，该类要求在 **macOS 主线程** 上创建 Display（Cocoa 限制），而 RAP 在 UI 线程中创建 Display，导致 `Invalid thread access`。

Cocoa 版 SWT 的日志中还会伴随警告：

```
WARNING: Display must be created on main thread due to Cocoa restrictions.
```

### 排查方法

检查 `WEB-INF/lib` 中是否存在桌面版 SWT jar：

```bash
ls tomcat-run/webapps/ROOT/WEB-INF/lib/ | grep -iE "swt|rwt"
```

正常情况下应**只有** RAP 的 RWT jar：

```
org.eclipse.rap.rwt-4.4.0.jar
```

如果出现 `org.eclipse.swt.cocoa.*` 或 `org.eclipse.swt.gtk.*` 等桌面版 jar，即为根因。

### 解决方案

删除桌面版 SWT jar，只保留 RAP 版：

```bash
rm -f tomcat-run/webapps/ROOT/WEB-INF/lib/org.eclipse.swt.cocoa.macosx.*.jar
rm -rf /tmp/rwt-resources tomcat-run/work/Catalina   # 清理缓存
# 重启 Tomcat
```

### 根本原因分析（为何会混入）

构建 WAR 包时，Maven 依赖解析将桌面版 SWT（用于 RCP/桌面客户端）一并打包进了 web 应用的 `WEB-INF/lib`。`assemblies/web/pom.xml` 应排除 `org.eclipse.swt.*` 的桌面版依赖，只保留 `org.eclipse.rap.rwt`。

### 关联文件

- `assemblies/web/pom.xml` — WAR 打包配置
- `rap/pom.xml` — RAP 模块依赖（应只依赖 RWT，不依赖桌面 SWT）
- `start-hop-web.sh` — 部署脚本（解压 WAR 到 `WEB-INF/lib`）
